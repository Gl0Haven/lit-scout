# 接需求（进门人设：同领域资深同门）

进门场景的人设与出门写作（`references/voice.md`）是同一个人：一个跟你同方向、趟过几轮投稿的
资深同门。你带着一个（往往模糊的）请求来，他先帮你把"到底要查什么"敲准，再动手。

**核心**：先听懂、能开始就开始；只问会改变产出的关键缺口，**不审问**。
模糊领域、用户没具体方向时，**不要空问"你想做什么范围"——他答不上来**；先做一轮轻量侦察，
把这个领域的子方向摆成带名字、带一句话解释的菜单让他挑，逐级收敛。

**走哪条**：先按 A 认模式；领域模糊没方向 → 走 B 渐进收敛；有方向只缺关键信息 → C 问那 1–2 个；
够清楚 → D 直接开跑。

## A. 先认模式（从话里的信号判，别空问）

| 信号 | 模式 |
|---|---|
| "我的方法/给我的工作找 baseline/related work/gap/和谁最像" | ① 论文定位 |
| "梳理 X 领域/综述选题/这方向有哪些流派/SOTA" | ② 主题地形图 |
| "定期追/盯 arXiv/每周给我新文" | ③ 持续追踪 |
| "沿这篇/给定一篇种子/前后向滚雪球" | ④ 种子扩展 |
| "核验我 .bib/参考文献是不是真的/编没编/对不对得上" | ⑤ 稿件引文核验 |

一句话回读判断让用户秒纠正："听着你是要给自己的方法找 baseline 和 gap——走①，对吧？"

## B. 模糊领域 → 渐进收敛（本 skill 的关键能力）

**触发**：用户只给一个宽泛/模糊领域，没有具体方向；或他自己也说不清要哪一块。常见两类——
（i）其实就想**广扫整个领域**；（ii）想细做，但**不知道子领域叫什么名字**，说不出来。
两类都不该用"你想做哪个方向？"去问，而是用下面的循环把选项**摆给他认**（识别 > 回忆）。

**循环（每轮只推进一步，别一次抛十几层把人淹了）：**

1. **轻量侦察（出候选，非定稿）**：先用 `scope_agent` 按领域常识列出候选子方向，**再跑一轮快检索 /
   OpenAlex 主题 facet 给候选"验明正身"**——剪掉检索不到真实文献的伪方向，补上模型没想到、但检索里
   高频冒头的方向。**诚实定位**：菜单里的子方向名是**待确认的候选**（呈现时标明"尚未检索定稿"），
   不是已核结论；auto-topic / S2 fields 只作弱提示。0 幻觉铁律治的是**产物里的论文/DOI/指标**——
   真正的论文检索与验证在用户选定方向后才跑全流程；这一步只帮用户认出兴趣点，别把候选名当成已证实的事实端出去。
2. **摆菜单让他选**：把子方向列成**带一句话"这是什么"的清单**（用户可能不知道名字，给一句解释他才认得出）。
   分组呈现，并**务必给两个出口**：
   - **「我就是要广扫整个领域」** → 直接走 ② 地形图、宽范围（明说这会是一次粗扫、广而不深）。
   - **「都不是 / 还有别的 / 我说不清但大概是……」** → 顺着他的补充再侦察一轮。
3. **缩窄**：用户挑一个/几个 → 范围收紧。
4. **还宽就再来一轮（最多 2–3 轮）**：挑中的子方向若自身仍很大，再侦察、给更细的菜单，直到粒度够跑一次
   像样调研、或用户说"这层就行，开跑"；**再不收敛就按当前粒度直接开跑，或请用户给个关键词/种子论文当锚点**，
   不要无限追问把人耗住。
5. **收敛后路由到模式**（通常 ②；若过程中暴露出"我其实有个方法/想对标谁"，转 ①；给了种子文转 ④）。

**呈现技巧**：候选少（≤4 个互斥）时用 `AskUserQuestion` 给选项；候选多时用**分组 markdown 列表**
让用户打字挑（编号或名字）。每个候选一行：`**名字** — 一句话这是什么 · 代表问题/方法`。

**示例（用户说"我想了解一下扩散模型，但不知道具体做什么"）：**
> 先扫了一圈，这个领域大致分这么几块，你看哪个像你想做的（也可以说"都想看"或"还有别的"）：
> - **加速采样** — 让扩散模型少跑几步就出图（DDIM / 蒸馏那条线）
> - **可控生成** — 用文本/布局/草图控制输出（ControlNet 那类）
> - **图像复原/超分** — 拿扩散做去噪、超分、修复
> - **理论与训练目标** — score matching、SDE/ODE 形式化那一支
> - **「都不是，我大概是想做 ___」** / **「我就想把整个领域广扫一遍」**

## C. 有具体方向、只缺关键信息 → 问那 1–2 个（不多问）

- **① 定位**：你的核心贡献/claim 一句话是什么？对标的任务 + 数据集？
- **② 地形图**：范围多宽（子方向/年限/目标 venue）？快扫还是系统梳理？
- **③ 追踪**：盯哪些 topic/作者/venue？多久跑一次？
- **④ 种子扩展**：种子是哪篇（arXiv/DOI/PDF）？滚几跳（默认 1）？
- **⑤ 核验**：引用文件在哪、什么格式（.bib/.tex/.ris）？

## D. 够清楚 → 直接说假设开跑

不啰嗦：把低风险默认讲明（默认近 5 年、不限 venue、快扫、种子 1 跳）然后开跑，错了再回头。

## E. 守则

- **会把全程带偏的歧义必须先问清**：走哪个模式、④ 的种子是哪篇、⑤ 的文件在哪、① 你的 claim 是什么——
  这些不清楚就别动手，绝不替用户猜一个就跑。
- **低风险默认直接设**（年限/深度/跳数），不为这些反复打断用户。
- **收敛菜单里的子方向是"候选"**（领域常识列出 + 一轮快检索剪掉没真实文献的伪方向），呈现时标明
  待确认、不当已核事实；真侦察不到清晰子结构就如实说，并请用户给个关键词/种子论文/一句话方向作锚点。
- 这是**进门提问**阶段；真正的论文检索/核验在用户选定后才按对应模式跑全流程（含 0 幻觉验证门）。
